Media Queries

1. Introduction to Media Queries:

Media queries in CSS allow you to apply different styles based on various factors such as screen size, device orientation, and resolution. They are crucial for creating responsive designs that adapt seamlessly to different devices and screen sizes.

2. Syntax and Structure:

Media queries begin with the @media rule, followed by a media type (e.g., screen, print) and optional media feature expressions enclosed in parentheses. For example: 
    @media screen and (max-width: 768px)
     {
         /* CSS rules for screens with a maximum 
         width of 768 pixels */
     }

3. Using Media Features:

Media features enable you to target specific characteristics of the user's device, such as width, height, orientation, and resolution. For instance: 

    @media (orientation: landscape) 
    {
        /* CSS rules for devices in landscape 
        orientation */
    }

4. Common Use Cases:

Media queries are commonly used to create responsive layouts, adjust typography and font sizes, modify image sizes and positioning, and hide or show elements based on screen size. For example: 
        @media screen and (max-width: 576px) {
            /* Adjust layout for small screens */
        }
    
        @media (min-resolution: 300dpi) {
            /* Apply high-resolution styles for devices 
            with high pixel density */
        }

5. Best Practices:

When using media queries, it's essential to start with a mobile-first approach by applying styles for smaller screens first, then progressively enhancing the design for larger screens. Additionally, test your design across various devices and screen sizes to ensure optimal responsiveness.
